Using Manifest Files with the ACDC Platform

Download and access study files securely using the Gen3 client and a manifest file.

What is a Manifest File?

A manifest file is a JSON file that lists the dataset files you've selected in the Study Explorer. It contains metadata like GUIDs and file names required to securely download those files using the Gen3 client.

This file is machine-readable and is typically named manifest.json.

How to Generate a Manifest

  1. Navigate to the Study Explorer.
  2. Select one or more studies or data files using the checkboxes.
  3. Click the Download Manifest button.
  4. A file named manifest.json will be downloaded to your computer.

How to Use the Gen3 Client

The Gen3 client is a command-line tool for securely downloading data from the ACDC platform using your manifest file.

  1. Install the Gen3 client: 
    pip install gen3-client
  2. Configure the client: 
    gen3-client configure --profile acdc --cred ~/.gen3/credentials.json --apiendpoint https://commons.heartdata.baker.edu.au
  3. Download files from your manifest: 
    gen3-client download-multiple --profile acdc --manifest-path manifest.json

For full instructions, see the official Gen3 Client documentation.

Troubleshooting Tips

  • Ensure you are logged in and have access to the selected study data.
  • Manifest files are only valid for a limited time; regenerate if expired.
  • Check your API endpoint is correctly configured to https://commons.heartdata.baker.edu.au.
